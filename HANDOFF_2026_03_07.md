## Architecture Notes

### Auth Flow
```
tokenStore.setTokens(access, refresh)  ← after login
    ↓
apiClient request interceptor  → Authorization: Bearer <access_token>
    ↓
401 response interceptor  → POST /v0/auth/refresh  → retry original request
    ↓  (if refresh fails)
notifyTokenExpired()  → clears tokens, fires callback → redirect to login
```

### Agent Playground Dual-Mode
```
DRY RUN (dryRun=true):
  sendMessage() → KernelSession (with history) → extract last agent reply → addMessage

LIVE EXECUTION (dryRun=false):
  runInSession() → KernelRun → setActiveRunId(run.id)
                                    ↓
  useRunStream(activeRunId) → SSE events → handleRunEvent
                                    ↓
  RUN_COMPLETED → extract outputs.response → addMessage → clear activeRunId
  RUN_FAILED → add system error message → clear activeRunId
```

### Real-time Event Manager
```
getEventManager()  ← singleton
  .subscribeRun(runId, callback)  → opens SSE via createRunStream()
  .subscribeAll(callback)  → receives all events across all runs

Auto-closes streams on RUN_COMPLETED / RUN_FAILED / RUN_CANCELED
Deduplicates: multiple subscribers to same runId share one SSE connection
```

### Deep-Link Routes
```
Workflow Studio:
  /builder/:workflowId  → BuilderRoute (hydrates uiStore.workflowId)
  /runs/:runId           → RunDetailRoute (passes initialRunId to RunsPage)

Agent Studio:
  /builder/:agentId      → BuilderRoute (hydrates uiStore.agentId)
  /playground/:sessionId → PlaygroundRoute (passes initialSessionId)

Board Studio (already existed):
  /boards/:boardId
  /boards/:boardId/cards/:cardId
  /boards/:boardId/release-packet
```
